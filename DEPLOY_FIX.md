# CloudBase 404 错误修复指南

## 问题原因
访问 `/test_1/` 路径时出现 404，说明文件路径配置不匹配。

## 解决方案

### 方法一：检查 CloudBase 控制台配置（推荐）

1. **登录 CloudBase 控制台**
   - 访问：https://console.cloud.tencent.com/tcb

2. **进入静态网站托管**
   - 选择你的环境
   - 点击「静态网站托管」

3. **检查部署路径配置**
   - 查看「部署路径」或「访问路径」设置
   - 确认是否配置了 `/test_1/` 路径

4. **两种修复方式**：

   **选项 A：修改部署路径为根路径**
   - 将部署路径改为 `/`（根路径）
   - 然后修改 `vite.config.js` 中的 `base: '/'`
   - 重新部署
   - 访问：`https://你的域名.tcloudbaseapp.com/`

   **选项 B：确认文件部署到 `/test_1/`**
   - 确保 CloudBase 配置的部署路径是 `/test_1/`
   - 确保 `vite.config.js` 中 `base: '/test_1/'`
   - 重新部署

### 方法二：检查实际部署的文件

1. **查看部署日志**
   - 在 CloudBase 控制台查看「部署历史」
   - 确认文件是否成功上传

2. **检查文件结构**
   - 确认 `dist` 目录下有 `index.html`
   - 确认文件已正确构建

### 方法三：使用根路径部署（最简单）

如果不需要子路径，建议部署到根路径：

1. **修改 vite.config.js**
   ```js
   base: '/',  // 改为根路径
   ```

2. **在 CloudBase 控制台**
   - 将部署路径设置为 `/`（根路径）
   - 或删除子路径配置

3. **重新部署**
   - 访问：`https://你的域名.tcloudbaseapp.com/`
